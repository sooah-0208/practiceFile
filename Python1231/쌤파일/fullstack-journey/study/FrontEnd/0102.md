```
import { useState } from "react"

const Page3 = () => {
    const [array, setArray] = useState([])
    const [txt, setTxt] = useState('')
    const submitEvent = e => {
        e.preventDefault()
        if(txt === ''){
            return(
                setArray([txt, ...array]),
                setTxt('')
            )
        }
    }
    return(
        <form onSubmit={submitEvent}>
            <input type="text" placeholder="글을 입력하세요."
            value={txt} onChange={e => setTxt(e.target.value)} 
            />
            <button type="submit">입력</button>
            <div>
                {
                    array.toReversed().map((v,i) => <p key={i}>{v}</p>)
                }
            </div>
        </form>
    )

}

export default Page3
```

form / input / button 등은 값을 입력받아야 하기 때문에
defaltValue + readOnly를 쓰면 안됨. 기본값을 읽기만 할 수 있으니까

- `e.preventDefault()`
form은 기본적으로 action(event)을 가짐 어딘가로 가려고 하는 action. 그래서 뒤에 어떤 이벤트 발생 시 이동할건지 이벤트 속성을 넣어줌
이 이동을 막으려고 `e.preventDefault()`를 씀 -> 얘는 react아니고 js문법
이동이 막혔기 때문에 뒤에 if문도 쓸 수 있음
이동을 안 막으면 뒤에 코드들이 실행됐는지 아닌지 알 수 없어짐

` if(txt === '') return`
txt가 비어있을 때 아래 setArray, setTxt를 실행하지 않으려고 return으로 막은거임
->return의 특성: if가 해당되면 return 이후의 코드는 실행하지 않음

`...array` => 깊은복사
array[]에 들어있는 값들을 인덱스 값으로 하나씩 꺼내 가지고있는 방법
array[1, 2, 3]
inArray[...array] ==> inArray[array[0](0), array[1](=2), array[2](=3)]
이거라서
inArray[0] = 5 하면
inArray[5, 2, 3]이 됨

`setTxt('')` input에 적혀있는 글씨가 useState 때문에 계속 저장되어 있기 때문에 빈칸으로 초기화 시키는 역할
이거 없으면 input에 글씨 적고 입력 눌러도 글씨가 그대로 있음. 일일히 지웠다 새 내용 입력해야함.

toReversed() 배열의 순서를 반전시켜줌
`setArray([txt, ...array])` 이 배열이 전해주는 순서를 for문으로 꺼내 일일히 바꿀 수 없으니 리버스 시켜 오름/내림차순 정리
-> 이걸 오름차순/내림차순 버튼 이벤트에 넣어 주면 됨
