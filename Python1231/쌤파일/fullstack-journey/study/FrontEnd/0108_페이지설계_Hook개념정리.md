## 1️⃣ useState는 언제 쓰는가 🧠
useState 는 **수정 모드가 필요한 곳에만 사용** (값 변경되는 곳)  
그래서 **변동 없고 읽기만 하면 되는 화면은 안 씀**

**예)**  
- 사용자가 입력할 때(form 존재)  
- 버튼 클릭으로 상태 변화  
- 변경 값이 화면에 즉시 반영돼야 할 때  

---

## 2️⃣ ?? vs || 차이 ⚖️

- `a ?? b` : a가 **null이나 undefined**일때만 b 사용  
  => 값이 존재하지 않을 때만 대체가 됨  

- `a || b` : a가 **falsy**이면 (false, "", 0, null, undefined, NaN)  
  => 초기값 0이나 빈문자열일때 문제될 수 있음  

👉 **??가 더 안전함**

## 3️⃣ form / button 기본 규칙 📝

### ✔ submit 버튼
- form에 `onSubmit` + `button type="submit"`
- form 당 **Only 한개!**  
  (이 버튼으로만 값 저장해 보낼 수 있도록)
- 엔터키로도 제출됨  

### ✔ 나머지 버튼
- **무조건 `type="button"`**

## 4️⃣ CRUD란? (실무 기본 용어) 🏗️
CRUD = 데이터가 거치는 **4가지 기본 동작**

| 글자 | 뜻 | 예시 |
| --- | --- | --- |
| **C** | Create | 새 글 작성 |
| **R** | Read | 글 보기 |
| **U** | Update | 글 수정 |
| **D** | Delete | 글 삭제 |

👉 게시판, Todo, 쇼핑몰, 관리자 페이지  
다 이 **4개 조합**으로 이루어져있다
