# 1️⃣ 문서 구조 태그 (Layout / Semantic)
## 👉 페이지의 큰 구조를 나눔
| 태그          | 설명               |  |
| ----------- | ---------------- | ----  |
| `<html>`    | HTML 문서의 시작      | 이 문서가 HTML임을 브라우저에 알림 |
| `<head>`    | 메타데이터, CSS, JS   | 화면엔 안 보이나 문서에 필요한 정보 입력(css연결, 문자인코딩, 검색엔진 정보, 페이지 제목) |
| `<body>`    | 화면에 보이는 내용       | 사용자 눈에 보이는 모든 정보 입력(하위 태그들) |
| `<header>`  | 상단 영역 (로고, 제목 등) | |
| `<nav>`     | 메뉴 / 내비게이션       | 페이지 이동용 링크를 묶을 때 (a href) |
| `<main>`    | 페이지의 핵심 내용       | menu, footer제외 딱 1개의 중심컨텐츠 // nav header footer은 못넣음 |
| `<section>` | 주제별 구역           | |
| `<article>` | 독립적인 콘텐츠         | 뉴스기사, 블로그 글, 게시글 등 |
| `<aside>`   | 사이드바, 보조 내용      | 광고, 인기글 등 |
| `<footer>`  | 하단 영역 (저작권 등)    | |

# 3️⃣ 링크 & 미디어 태그
## 👉 이동, 이미지, 영상
| 태그         | 설명        | |
| ---------- | --------- | --- |
| `<a>`      | 링크        | 페이지 이동 / 파일 열기 / 위치이동 |
| `<a>` 속성    | `href` / `target="blank"` / `download`|이동할 주소 / 새 창 / 파일 다운로드|
| `<img>`    | 이미지 **닫는 태그 없음**       | `src`이미지 경로 / `alt`이미지 설명 |
| `<audio>`  | 오디오       |`controls` 재생버튼 / `autoplay`자동재생 / `loop`반복 |
| `<video>`  | 비디오       | 오디오와 동일+ `muted`(자동재생 시 필수) |
| `<source>` | 미디어 파일 경로와 형식 알려줌    | audio/video/picture하위 태그|
| `<iframe>` | 다른 웹페이지/영상 삽입 | |

# 5️⃣ 폼(Form) 태그 
## 👉 입력, 버튼, 사용자 상호작용
| 태그           | 설명      | |
| ------------ | ------- | --- |
| `<form>`     | 입력 폼    | `action`데이터 전송 / `method="(`get` / `post`)"전송 방식
| `method=""`|기본값(즉 생략가능)=get - 주소값을 복사해 공유할 때(영상/지도 등) | post - 주소에 민감한 정보를 배포하지 않을 때 사용(로그인 등) / 백엔드 필요
| `<input>`    | 입력창     | `radio` 여러 항목중 하나는 필수 선택 / 'check box`는 여러항목 선택 가능
| `<input>` | `name=""` | 이게 있어야 링크에 내용이 같이 이동함 |
| `<input>` | `type=""' | 입력할 내용 지정(date 날짜/text 글/color-색상표/file-파일선택 등)
| `<textarea>` | 여러 줄 입력 | 창 크기 조절 가능함
| `<button>`   | 버튼      |
| `<select>`   | 드롭다운    | 연속으로 제어하려면 java필요(ex)연/월/일 선택하는 방식)
| `<option>`   | 선택 항목   |
| `<label>`    | 입력 설명   | input의 id와 연결됨(필수)/라벨의 내용을 클릭해도 input 입력창으로 넘어감
| `<fieldset>` | 폼 그룹 묶는 용도   | 자동으로 테두리 생성됨
| `<legend>`   | 그룹 제목   | 'fieldset'으로 만든 테두리에 제목으로 들어감
| `placeholder=""` | 입력창에 넣을 내용을 알려주는 용도 | 아이디/비번 등 입력 전에 칸에 쓸 내용을 알려줌
| `required` | 무조건 내용 입력이 필요할 때 | 경고창이 뜨며 다음 화면으로 이동하지 않음(경고창의 내용 변경을 원하면 java로 해야함)

# ✅ 인라인 vs 블록 태그 (중요!)
## 🔹 블록(Block)  
한 줄 전체 차지
예: `<div>` `<p>` `<h1>` `<section>` `<nav>`  
### div
- Block-level 컨테이너(블록 단위 요소)
- 웹 페이지에서 섹션이나 그룹을 붂는 용도
- 의미적 역할은 없음 => 단지 css(스타일링)나 레이아웃을 위해 사용

## 🔹 인라인(Inline)  
내용만큼만 차지
예:`<span>` `<a>` `<strong>` `<em>` `<img>`

# 🔔 한 줄 요약
구조: header, nav, main, section
글자: h1, p, span
입력: form, input
표: table
동작/디자인: script, link
