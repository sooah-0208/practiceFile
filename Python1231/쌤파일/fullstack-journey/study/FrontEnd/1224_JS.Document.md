## 1️⃣ JS 파일 연결하기

### 📄 JS 파일 생성 후 HTML에 연결
```html
<head>
  <script src="js파일명"></script>
</head>
```

## 2️⃣ JS가 잘 연결되었는지 확인하는 방법
콘솔을 켬 ->   
`document.getElementsByTagName('section')[0];` ->  
라고 입력했을 때 내가 원하는 부분이 선택된 게 보이면 연결 된 것! 


### ❓ 왜 `[0]`이 붙을까?
- body 안에 같은 태그가 **여러 개 있을 수 있음**
- 그래서 기본적으로 **배열(Array) 자료형**
- 그중 첫 번째 요소를 가져오기 위해 `[0]` 사용



## 3️⃣ document에 요소 넣는 방법

```
// js파일
function 이름(){
let 요소 = "";(문자열) / 숫자열
document.getElementsByTagName("section")[0].inner(형식/HTML이랑 TEXT있음) = 요소명;
}
```

- `innerHTML= 요소` : 태그 안에 내용을 HTML형식의 요소로 바꾸는 속성
- `innerTEXT= 요소` : 태그 안에 내용을 TEXT형식의 요소로 바꾸는 속성

## 🤔 어? 안 나오는데요?

### ❗ 이유
- 코드는 **위에서부터 순서대로 실행**
- script가 태그보다 위에 있으면
- 아직 태그가 없는 상태에서 실행 → 오류 발생

### ✅ 해결 방법
1. onload
```html
<body onload="함수이름()">
```
 📌 태그 안에 onload란?
- 태그 안의 **모든 태그가 실행된 후**
- `onload="얘"` ← 이 함수 실행하겠습니다 라는 의미
- 가장 부모 태그인 body에 보통 넣음

2. body 태그 뒤에 script 넣기
- 코드는 위에서 부터 읽으니까 body 이후에 넣으면 body를 다 읽고 scritp를 실행하기 때문에 정상적으로 실행 됨

#### 🤔WHY? 그럼 onload를 주로 쓰나요?
🔥 이유 1. “이 함수는 첫 화면용이다”를 명확히 보여줌  
`<body onload="init()">`이 한 줄만 봐도:  
“아 이 페이지는 init 함수부터 시작하는구나” 알 수 있음.  
script 맨 아래에 있으면 어떤 게 시작 함수인지 바로 안 보임  
  
🔥 이유 2. JS 파일 위치랑 무관해짐  
🔥 이유 3. “시작 함수”를 하나로 관리 가능  
```
//js파일
function init(){
    clean();
    magicInit();
    bindEvent();
}
//html 파일
<body onload="init()">
```

이렇게 해두면 시작 로직이 무조건 `init()` 있다는 걸 알 수 있음
🔥 이유 4. 협업 / 가독성  
실무에서는: HTML → 구조  /  JS → 로직  이렇게 역할 분리함
`<body onload="init()">` 이것만 봐도 아 js파일에 init이라는 시작점이 있구나 알 수 있음
