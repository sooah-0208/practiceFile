# ⚛️ React JSX 문법 & 이벤트 핸들링 정리

## 1️⃣ 속성(Attributes) 작성 규칙
React의 JSX는 HTML이 아닌 **JavaScript**를 기반으로 함. 따라서 아래 규칙 준수 필수.

* **모든 Tag를 닫아줌**: html에서는 `<input>`만 해도 됐지만, jsx는 `<input />`으로 닫아줘야함
* **CamelCase 사용**: HTML에서 `-`로 연결하던 속성명은 대문자로 붙여서 작성.
    * 예: `on-click` ➡️ `onClick`, `tab-index` ➡️ `tabIndex`
    * ⚠️ 미준수 시 React에서 속성 인식 불가.
### ⚛️ HTML vs JSX 속성명 변경 리스트
| HTML 속성 | JSX (React) 속성 | 이유 |
| :--- | :--- | :--- |
| `class` | **`className`** | 자바스크립트의 `class` 키워드와 충돌 방지 |
| `for` | **`htmlFor`** | 자바스크립트의 반복문 `for`와 충돌 방지 |
| `tabindex` | **`tabIndex`** | 모든 속성은 **CamelCase**(카멜 케이스)로 작성 |
| `onclick` | **`onClick`** | 이벤트 핸들러는 CamelCase 작성이 필수 규칙 |
| `onchange` | **`onChange`** | 위와 동일 (CamelCase 규칙) |
| `readonly` | **`readOnly`** | CamelCase 규칙 적용 |
| `maxlength` | **`maxLength`** | CamelCase 규칙 적용 |
| `colspan` | **`colSpan`** | CamelCase 규칙 적용 (표 레이아웃 시 사용) |
| `autofocus` | **`autoFocus`** | CamelCase 규칙 적용 |

## 4️⃣ 이벤트 핸들링 (Event Handling)
기존 HTML 방식(`onclick="location.href='...' "`) 사용 시 화면은 정상 출력되나 콘솔 에러 발생.

### ✅ 올바른 변경 방법
1. **함수 정의**: 컴포넌트 최상단에 실행할 함수 정의.
   - `const onclick = () => { location.href = 'Select.html'; }`
2. **Tag 적용**: 중괄호 `{ }`를 사용하여 함수 객체를 직접 전달.
   - `<tr onClick={onclick}>` 

### 🔍 중괄호 `{ }` 사용 이유
JSX에서 `{ }`는 **자바스크립트 표현식** 사용을 의미.
* **HTML**: `"onclick()"`이라는 **문자열(글자)** 전달.
* **JSX**: `onclick`이라는 **실제 함수 객체**를 직접 전달.

####  자주 하는 실수 🚨
❓ `{onclick()}` 처럼 괄호를 붙여서 전달하는 경우  
* **의미**: "페이지 로딩 즉시 이 함수를 실행하라"는 명령.
* **문제점**: 클릭 시 이벤트가 발생하는 것이 아니라, **렌더링과 동시에 함수가 실행**됨.
* **해결**: 괄호 없이 함수 이름만 작성(`{onclick}`)하거나, 인자가 필요한 경우 익명 함수(`{() => onclick(값)}`) 사용.

#### 🛠 활용 예시 (중괄호 표현식)
```jsx
// 1. 계산식 활용
<p>결과: {1 + 2 + 3}</p>

// 2. 삼항 연산자(조건문) 활용
<button>{isLoggedIn ? "로그아웃" : "로그인"}</button>

// 3. 익명 함수 활용 (함수 즉시 정의)
<button onClick={() => console.log("클릭됨!")}>버튼 클릭</button>

## 5️⃣ map() 함수를 이용한 반복 렌더링
배열 데이터를 순회하며 동일한 JSX 구조를 반복 생성할 때 사용.

```
 <tr class="cursor-pointer" onclick="location.href = 'Select.html';">
	        <td>스티븐</td>
	        <td>jobs@shellfolder.com</td>
	        <td>2023-02-28</td>
	      </tr>
	      <tr class="cursor-pointer" onclick="location.href = 'Select.html';">
	        <td>에브릴</td>
	        <td>lavigne@shellfolder.com</td>
	        <td>2023-02-27</td>
	      </tr>
```
👆이런 애들

### 💡 기본 구조 분석

```
  const arr = [
        {"name" : "스티븐", "email" : "jobs@shellfolder.com", "regDate" : "2023-02-28"},
        {"name" : "에브릴", "email" : "lavigne@shellfolder.com", "regDate" : "2023-02-27"},
    ]

 <tbody>
            {
            arr.map( (v, i)=>{
                return(
                <tr style={stlyes} onClick={onclick} key={i}>
                    <td>{v.name}</td>
                    <td>{v.email}</td>
                    <td>{v.regDate}</td>
                </tr>)
            })
            }
	      
	    </tbody>
```

- **`v` (value)**: 현재 순회 중인 배열 내의 **데이터 객체**. (예: `v.name`으로 상세 항목 접근)
- **`i` (index)**: 현재 데이터의 **순서 번호** (0부터 시작).
- **`key` 속성**: 리액트가 리스트 항목을 식별하기 위한 고유 값. 
    - 중복되지 않는 고유 ID를 권장하나, 없을 경우 순서 번호(`i`) 사용 가능.

### 🔍 왜 `{v.name}` 인가?
`v`는 객체 전체를 의미하므로, 점 연산자(`.`)를 사용하여 객체 내부의 특정 키(Key) 값에 접근하여 출력함.

### ⚠️ Key 설정 시 주의사항
* **Index 사용 지양**: `key={i}`와 같이 배열 인덱스를 사용하는 것은 권장되지 않음. 데이터 추가/삭제/정렬 시 리액트가 항목을 정확히 식별하지 못해 성능 저하나 예기치 못한 버그를 유발할 수 있음.
* **고유 ID 권장**: 데이터 자체에 포함된 `id`와 같은 고유 식별자를 `key={v.id}` 형태로 사용하는 것이 정석.

```
javascript
// 권장되는 데이터 구조 예시
const arr = [
  { id: 101, name: "스티븐" ... },
  { id: 102, name: "에브릴" ... }
];

// 적용 예시
<tr key={v.id}> ... </tr>
```

## 💡 Arrow함수 팁
`() => {}` : 내가 할 일을 {}안에 줄게~  
`() => ()` : 이 덩어리를 통째로 줄게 출력해!
👇 아래처럼 쓸 수 있음

```
// 괄호 자체가 return을 의미함
arr.map((v, i) => (
  <tr key={v.id}>...</tr> 
))
```
